The error `'NoneType' object has no attribute 'addObject'` occurs because there is no active document open in FreeCAD when the script runs. `FreeCAD.ActiveDocument` returns `None`, and you cannot add an object to `None`.

Here is the corrected code. I added a check to create a new document if one doesn't exist.

```python
import FreeCAD
import FreeCADGui

# Fix: Check if there is an active document. If not, create a new one.
if FreeCAD.ActiveDocument is None:
    FreeCAD.newDocument()

# 1. Create the Sphere
# Define the radius (e.g., 10 mm)
sphere = FreeCAD.ActiveDocument.addObject("Part::Sphere", "MySphere")
sphere.Radius = 10

# 2. Recompute to update the geometry
FreeCAD.ActiveDocument.recompute()

# 3. Fit the view to see the whole object
FreeCADGui.SendMsgToActiveView("ViewFit")

# 4. Set Isometric View
FreeCADGui.ActiveDocument.activeView().viewIsometric()
```

### Explanation of the fix:
*   **The Issue**: The original script assumed a document was already open. In FreeCAD, if you run a macro without an open file, `FreeCAD.ActiveDocument` is `None`.
*   **The Solution**: The lines `if FreeCAD.ActiveDocument is None: FreeCAD.newDocument()` ensure that there is always a place (a document) to put the sphere before creating it.
